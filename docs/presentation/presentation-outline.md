# Paper Proposal Presentation Outline

## Goal

Present the project as a symbolic benchmark for music theory and computational musicology, with a clean standard benchmark workflow and an optional explanatory layer for paper-oriented analysis.

## Slide outline

1. Title and project framing
2. Motivation: why a symbolic music-theory benchmark
3. Task families and benchmark outputs
4. Current V1 scope
5. Planned extensions and not yet final theoretical coverage
6. System architecture
7. Standard benchmark workflow for non-expert users
8. Canonical prediction schema
9. Automatic evaluation and result products
10. Musicology layer: standard vs explanatory modes
11. Use-case and user workflow views
12. Sequence and component views
13. Paper framing, contributions, and next steps

## Core messages to preserve

- the benchmark is already usable and formally evaluable
- the benchmark is intentionally constrained in V1 for reproducibility
- the theoretical scope is not yet final
- the project supports both practical testing and paper-oriented interpretation
- `prediction_explanation` is supplementary and should not destabilize the standard benchmark
